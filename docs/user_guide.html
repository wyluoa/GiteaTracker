1. 系統概觀

Gitea Tracker 是內部網頁工具，取代原本用 Excel 管理 QC Script Request 的流程。每個題目（Issue）依 ISO 週次分組，跨多個 Node（工作階段）以狀態追蹤到結案。

1.1 基本名詞

名詞說明
Issue / Topic 一筆 QC 請求。每題有獨特的題號，例如 301、P260、CN49、SNPS61。
Node 工作階段（A10、A12、A14、N2、A16、N3、N4/N5、N6/N7、000、MtM 等）。同一題會跨多個 Node。
Cell 「題目 × Node」的交集格子。每格有狀態（State）、Check-in 日期、備註。
Week（週） 題目依建立當週的 ISO week 分組。格式如 wk612 = 2026 年第 12 週。
Red Line Admin 設定的死線週。Red Line 之前的題目若仍為 UAT 或 TBD 會變紅警示。
Group Label 非週次分組的特殊標籤（例如「強身健體系列」）— 整組題目以獨立 card 顯示，不依週次分組。
On Hold 暫緩中的題目，顯示在 Tracker 下方可折疊區塊。

1.2 角色

角色權限
未登入訪客可瀏覽 Tracker 和 Closed（read-only）。
一般使用者登入後可編輯自己被分配到的 Node 的 cells，新增 comment / meeting note，上傳附件。
Manager 2026-04 額外可把任一 cell 改為 Unneeded。
Super User（admin） 所有編輯權限。可改狀態為 Done、建立題目、關單 / 重開、管理使用者、Excel 匯入等。

2. 登入與帳號

2.1 不登入也能看

Tracker 和 Closed 頁面是公開的（read-only）。直接瀏覽：

http://<server>/GiteaTracker/tracker

Read-only 模式 未登入時可瀏覽所有資料，但無法編輯、留言或關單。右上角會有「登入」按鈕。

2.2 註冊與啟用

  1. 導覽列點 登入 → Register。
  2. 填帳號、Email、顯示名稱、密碼（至少 6 字元）。
  3. 送出後帳號為 pending，需 admin 審核啟用才能登入。
  4. 啟用後用相同帳密登入即可。

2.3 忘記密碼

  1. 登入頁點 忘記密碼。
  2. 輸入註冊時的 Email，系統會寄送重設連結（Admin 需先設好 SMTP）。
  3. 開信點連結、輸入新密碼，完成重設。

2.4 修改密碼

登入後點導覽列顯示名稱旁的 改密碼。

4.2 凍結欄位標題 2026-04

題目多的時候，主表 / 強身健體 / On Hold / Closed 各自的欄位標題（Topic、Owner、A10、A12…）會黏在頁面頂端。捲到下方也不會迷失對應到哪個 Node。

4.3 格子 hover tooltip 2026-04

滑鼠停在任一格約 1 秒會顯示 A10: Dev 這種 node + 狀態 tooltip，Th 上則顯示完整 node 名。

4.4 搜尋與篩選

工具列提供：

  • 文字搜尋 — 對題號 / Topic / JIRA / Path 做包含比對。
  • Owner 下拉 — 僅顯示某位 Owner 的題目。
  • Node 下拉 — 限定某 Node。
  • 狀態下拉 — 限定某狀態（含「空白」）。
  • 週次範圍 — 例：202610 到 202615（格式 YYYYWW）。

4.5 進階篩選（AND / OR 可切換） 2026-04

點工具列「進階篩選」展開面板，最多 3 組「Node + 狀態」條件。新增的 條件之間 下拉可切：

模式行為
全部符合 (AND) 所有條件都命中的題目才保留。較嚴格。
任一符合 (OR) 只要命中任一條件就保留。較寬鬆，常用於「A10 Done 或 A12 Done 都算」。
小技巧 單一條件時 AND 和 OR 結果一樣；條件之間的 AND/OR 差異要兩條以上才有感覺。

4.6 週分組 + Red Line

題目依 ISO 週分組，標頭例如「▼ wk612 (3 題)」，點標頭可折疊整組。 Admin 設定 Red Line 後，不同週之間會畫紅色虛線。Red Line 之前還在 UAT 或 TBD 的 cell 會變 紅色警示逾期。

4.7 強身健體系列 / 特殊分組

帶 group_label 的題目會獨立成一張 card（例如「強身健體系列」），不依週分組。 點 card 標頭可折疊。欄位對齊主表，凍結標題也獨立運作。

4.8 On Hold

頁面底部紅色 card「⏸ On Hold (N)」，放暫緩中的題目。預設折疊，點標頭展開。

4.9 批次更新

  1. 勾選題號前的 checkbox，可勾多題。
  2. 畫面上方出現批次列，選 Node 和 狀態。
  3. 點「批次套用」→ 跳出確認視窗。
  4. 必填 備註（說明為何批次改）→ 確認即完成。

4.10 標記已讀 / 匯出 Excel

  • 標記已讀：上次瀏覽後被改過的資訊會有淡黃底色，點這個鈕一次清掉。
    2026-04-21 黃底現在分兩類，彼此獨立：
    • Cell 級 — 某個 Node 格子的狀態 / check-in 日期 / 短備註被改過。
    • 欄位級 — 整題的 Topic / Owner / JIRA / UAT Path 被改過；只亮被改的那一欄，不是整列。
    例如只有 JIRA 改過 → 只有 JIRA 欄亮黃底，其他欄位照常。
  • 新帳號第一次進頁面 2026-04-21： 不會看到滿螢幕黃底。系統在你帳號啟用時會自動把基準點設為「帳號開通的當下」，之後才發生的變動才算「新」。
  • 舊帳號：若從來沒按過「標記已讀」可能看不到任何黃底；按一次建立基準點就正常了。
  • 匯出 Excel：下載目前所有 ongoing + on hold 題目（含所有 node 狀態）到 .xlsx。

5. 狀態定義

Chip狀態意義
Done Done 此 Node 已完成。僅 super user 可設為 Done 2026-04。
UAT✓ UAT DoneUAT 驗證已完成。
UAT UAT等待或正在做 UAT 驗證。
Dev Developing開發中。
TBD TBD待定 / 尚未動工。
Unneeded Unneeded 此 Node 不適用。僅 super user 或 Manager 可設為 Unneeded 2026-04。
· （空白）尚未設定任何狀態。

5.1 狀態變更規則 2026-04

  • 狀態真的有變時（例如 UAT → Done），更新說明欄位變為必填，會有黃底提示。
  • 只改 Check-in 日期 / 短備註 / 留言而狀態不變時，更新說明仍為選填。
  • 若某 cell 的 目前狀態 是你沒權限改到的（例如一般使用者看到 Done），側面板狀態下拉會標 (僅管理員可改)。

6. Side Panel 與格子編輯

點 Tracker 任一 cell 會從右邊開 Side Panel。

6.1 編輯區

若你有此 Node 的編輯權限，會看到：

  • State 下拉 — 選新狀態；有些選項依角色隱藏。
  • Check-in Date — 預定 / 實際 check-in 日期。
  • Short Note — 顯示在 cell 內的短備註。
  • 更新說明 — 留在 Timeline 的說明。狀態改變時必填。
  • 附件 — 上傳 PNG / JPG / PDF，單檔上限 5 MB，最多 3 個。

點 儲存 後，主表該格即時更新。

只想補說明或圖片、不改狀態也可以存 2026-04-21 以前若 cell 已經是 Unneeded（或任何狀態沒有要動時）， 你在「更新說明」寫字或上傳附件按儲存，會被靜默忽略 — 系統以為沒東西變。 現在這種情況會被存成該 Node 的 Timeline 留言（含附件），之後用 Side Panel 的 Timeline 就能回顧。

6.1.1 編輯 Owner / Topic / JIRA / UAT Path 2026-04-21 （super user）

Side Panel 上方「編輯 Owner / Topic / JIRA / Path」按鈕會展開一個小表單：

  • Owner — 純文字，輸入後點更新。
  • Topic — 題目標題。
  • JIRA（新增）— 多行 textarea，可換行輸入多個 ticket，例如 JIRA-123
    JIRA-456    。
  • UAT Path — 多行 textarea，每行一條路徑。

Topic / JIRA / Path 在同一個表單，按一次「更新 Topic / JIRA / Path」一起存。 存完主表的對應欄位會自己亮黃底提示有改動（per-column highlight）。

權限 只能編輯被分配給你 group 的 Node。若 Side Panel 顯示「Read-only」，請找 admin 調整群組。 Owner / Topic / JIRA / Path 這幾個 issue 層級欄位只有 super user 能改。

6.2 Timeline（歷史紀錄）

  • 預設只顯示 目前 Node 的狀態異動，加上 General Comment 和 Meeting Note。
  • 點 View All 看整題跨所有 Node 的完整歷史。
  • 上方型別下拉可過濾 狀態異動 / Comment / Meeting Note。

6.3 Comment / Meeting Note

  • + General Comment：一般留言，可加附件。
  • + Meeting Note：會議記錄，可指定所屬週次。

6.4 整行套用（Row Apply）

題號旁邊的鉛筆 ✎ 圖示（super user）會展開一列勾選多個 Node 一次改到同一狀態，加上共用的更新說明。

7. 關單流程

7.1 全部完成 → ✓關（綠）

當題目的所有 Node 狀態都是 Done 或 Unneeded 時，題號旁出現綠色 ✓關 按鈕。點 → 可選填 Closed Note → 確認關單。

7.2 匯入建議關題 → ✓關?（黃） 2026-04

Excel 匯入時若 Ongoing 分頁某題 Status 欄填了 Closed，系統不會自動關單， 而是標為「建議關題」：題號旁出現黃色 ✓關? 按鈕。Admin 人工確認後再關。

Dashboard 也有獨立的「匯入建議關題」卡片，點 看題號 可展開所有待確認的題號。

7.3 Reopen

Closed 頁面的 super user 可對已關題目按 Reopen，填原因後移回 Ongoing。

8. Closed 頁面

所有已關題目，依週次分組（同 Tracker 排版）。免登入可看（read-only）。

  • 搜尋條支援題號 / Topic / JIRA。
  • 每列顯示最終狀態 + Check-in + 短備註 + Closed 日期 + Closed Note。
  • 分頁下方分頁列（每頁 50 題）。
  • Super user 可點 Note 欄 inline 編輯 closed note。

9. Dashboard 儀表板

需登入。提供統計卡片 + 趨勢圖表 + 洞察表。

9.1 統計卡片

卡片邏輯
Ongoing進行中題目總數。
Ready to Close 所有 node 皆為 Done 或 Unneeded 的題目數。卡片有 ? 提示完整計算邏輯，點「看題號」展開符合題號（連到 Gitea） 2026-04。
匯入建議關題 2026-04 有 pending_close flag 的題目數。同樣有 tooltip 和「看題號」展開。
On Hold暫緩題目數。
Closed已關單總數。
Closing Rate（含 MtM）Closed / (Ongoing + On Hold + Closed)。
Closing Rate（不含 MtM） 同上公式，但 MtM 為 Done / Unneeded 的題目視同「已結束 MtM」提前計入完成。

9.2 圖表

  • Topic Cumulative Trend — 每週 TBD / Dev / UAT / Close 的堆疊長條。 右上有時間範圍選單：YTD / 近 30 週 / 近 50 週 / 全部 2026-04。
  • Closing Rate Trend — 逐週關單率折線；若 admin 有輸入手動 trend data 優先用它。 同樣有時間範圍選單。

9.3 Per-Node 統計區塊 2026-04-21

Dashboard 下半會看到 UAT / TBD / Dev 三組 per-node 卡片，各自計算該 Node 目前被標記為該狀態的題數：

  • UAT per-Node — cell 狀態為 UAT 或 UAT✓ 的題數。
  • TBD per-Node — cell 狀態明確標為 TBD 的題數。
  • Dev per-Node（新）— cell 狀態為 Dev 的題數。對稱 TBD 那塊，讓 node 負責人一眼看到自己手上還在開發階段有幾題。

9.4 TBD / Dev 統計的分類邏輯 2026-04-21

Dashboard 最上方的 Topic Cumulative Trend 堆疊長條，把每題歸到一個 phase：

優先Phase條件
1Close題目狀態 = closed。
2UAT 任一 node 為 UAT / UAT Done。
3TBD 任一 node 明確標為 TBD。
明確 tbd 是管理者刻意下的「尚未定案」記號，優先於 developing。 所以 1 個 cell tbd + 1 個 cell developing 也算 TBD。
4Dev 其他全部（預設 bucket）。
包含：全空白題（還沒動工）、只有 developing 的題、全 done/unneeded 但還沒正式關單的題。

9.5 洞察表

  • Almost Done — 只剩 1-2 個 Node 未完成的題目。
  • Stale Topics — 90 天以上沒人動的題目。
  • Per-Node Stats — 各 Node 的 UAT / TBD 題數（Red Line 之前單獨再算一次）。
  • Bottleneck / Weekly Velocity — 瓶頸分析與每週進度。

10. Calendar 行事曆

月曆格顯示每日哪些題目 × Node 有 Check-in Date。箭頭可切換月份。需登入。

10.1 可上線清單 2026-04

Super user 在 Calendar 會看到一張新 card「可上線清單」，列出 Check-in Date ≤ 今天但狀態還不是 Done / Unneeded 的 node，方便盯進度。

  • 超過今天的顯示「逾期 N 天」。
  • 剛好今天 due 的顯示「今日」badge。
  • 可直接按 ✓ Done 按鈕快速標完成（僅 super user）。

11. UAT Path 多行輸入 2026-04

11.1 顯示

同一題可有多條測試路徑，資料庫用 \n 分隔，主表每條路徑一行顯示。

11.2 編輯

Side Panel 和「+ 新增題目」的 Path 欄位都是 textarea，可直接多行輸入（等寬字體）。

11.3 複製

點主表 Path cell，整段含換行一併進剪貼簿。貼到 vim / Excel / Notepad 都會是真正的多行。

12. FAQ 常見問題

Q：我完全不能編輯任何 cell？

帳號要是 active（admin 啟用）、且被分配到該 Node 所屬的 group。找 admin 幫忙檢查。

Q：Path 要複製怎麼做？

直接點主表 Path cell 即可，右下角跳「✓ 已複製」toast。多行也會整段複製（含換行）。

Q：黃色底的 cell / 欄位是什麼？

自你上次瀏覽後有被改過的資訊。工具列按 標記已讀 一次清掉。 2026-04-21 之後黃底分兩種，獨立判斷：

  • Cell 級：某個 Node 格子（state / check-in / short note）被改過 → 該格亮。
  • 欄位級：該題的 Topic / Owner / JIRA / UAT Path 其中一欄被改過 → 只亮那一欄，不整列都亮。

若你從沒按過「標記已讀」（舊帳號），可能完全看不到黃底 — 按一次建立基準點就好。 新帳號系統在啟用當下已自動設好基準點，不需要你先按。

Q：題號怎麼變成連到 Gitea？

由 admin 在 Admin > Gitea URL Mapping 設規則：

  • P260 → prefix P，開 perl_scripts repo issue #260
  • CN49 → prefix CN，開 CN_TEAM repo issue #49
  • SNPS61 → prefix SNPS，開 Synopsys repo issue #61
  • 301 → 純數字，開 QC_requests repo issue #301

Q：切 dark / gray 後畫面沒改變？

按 Ctrl + Shift + R 硬刷一次，因為公司代理可能快取舊 CSS。 做一次即可，之後系統自動接管。

Q：我按了「關」但題目沒消失？

關單會讓題目從 Tracker 消失、移到 Closed 頁。重新整理 Tracker 再確認一次。

Q：怎麼看我之前改了什麼？

點該題任一 cell 開 Side Panel 的 Timeline；切 View All 可看整題跨所有 node 的完整紀錄（含誰 / 何時 / 改什麼 / 附件）。

Q：匯出 Excel 的欄位和原 Excel 一樣嗎？

一樣。重新匯入也能 round-trip（系統會用題號對現有題目做 diff preview，避免重覆建立）。

Q：JIRA 票號打錯了，要怎麼改？ 2026-04-21

2026-04-21 之前只能等 Excel 重匯才改得到。現在 super user 可以直接在 Side Panel 上方點 編輯 Owner / Topic / JIRA / Path 展開表單修改。存完主表 JIRA 那欄會亮黃底提示有改動。

Q：一個 cell 我只想補說明或上傳附件、不想改狀態，為什麼都不見了？

這是 2026-04-21 修好的 bug。之前若 cell 狀態沒變（例如已經是 Unneeded），單純打字 + 上傳附件按儲存會被系統忽略。 現在這種情況會自動存成該 Node 的一筆 Timeline 留言（含附件），Side Panel 的 Timeline 可以回顧。
