1. Overview

Gitea Tracker is a web application that helps the team track QC script requests across multiple workflow stages (Nodes). Each issue is assigned a unique number, organized by week, and tracked through a series of states until completion.

Key concepts:

TermDescription
Issue / TopicA work item (QC request) with a display number (e.g., 301, P260, CN49).
NodeA workflow stage (e.g., A10, A12, N2, MtM). Each issue goes through multiple nodes.
CellThe intersection of an issue and a node. A cell has a state, optional check-in date, and short note.
WeekIssues are grouped by the ISO week they were raised (e.g., wk612 = 2026 week 12).
Red LineA deadline set by admin. Issues above the red line that are still in UAT/TBD are highlighted in red.

2. How to Access

2.1 Without Login (Read-Only)

The Tracker and Closed pages are publicly accessible. You can view all issue data without logging in. The URL typically looks like:

http://<server>:<port>/GiteaTracker/tracker

Read-only mode Without login, you can browse all data but cannot edit cells, add comments, or perform any write operations. A "Login" button appears in the toolbar and navbar.

2.2 Register & Login

  1. Click Login in the navbar, then click Register.
  2. Fill in username, email, display name, and password (at least 6 characters).
  3. After registration, your account is in pending status. An admin must approve it.
  4. Once approved, you can log in and start editing.

2.3 Change Password

Click the Change Password link in the navbar (next to your username).

3. Tracker Page

This is the main working page. All ongoing and on hold issues are displayed in a table, grouped by week.

3.1 Table Layout

ColumnDescription
#Issue number. If Gitea URL mapping is configured, this is a clickable hyperlink to the Gitea issue.
TopicDescription of the issue.
OwnerPerson responsible.
Node columnsOne column per active node. Shows a colored state chip + optional check-in date + note.
JIRAJIRA ticket reference.
PathUAT file path. Click to copy to clipboard.

3.2 Sticky Header

When you scroll down, the column header row stays fixed at the top of the screen so you can always see which column you're looking at.

3.3 Search & Filter

The toolbar at the top provides:

  • Text search — matches #, Topic, JIRA, or Path.
  • Owner dropdown — filter by a specific owner.
  • State dropdown — filter issues that have any cell in a given state.
  • Week range — filter by week (e.g., 202610 to 202615).
  • Advanced filter — up to 3 conditions combining a specific Node + State (AND logic).

3.4 Week Groups & Red Line

Issues are grouped under collapsible week headers (e.g., wk612). Click a week header to expand or collapse that group.

If a Red Line is set, a dashed red line appears between weeks. Issues above the red line with UAT or TBD states are displayed in red to highlight urgency.

3.5 Batch Update

  1. Check the checkboxes next to the issues you want to update.
  2. A batch action bar appears. Select a Node and a State.
  3. Click Batch Apply, enter a reason, and confirm.

3.6 Mark All Read

Cells updated since your last visit are highlighted in yellow. Click Mark All Read to clear these highlights.

5. Side Panel & Cell Editing

Click any cell in the Tracker table to open the Side Panel on the right.

5.1 Cell Editing

If you have permission to edit this node, you will see:

  • State dropdown — select the new state.
  • Check-in date — planned or actual check-in date.
  • Short note — a brief note displayed inside the cell.
  • Update description — optional text recorded in the timeline.
  • Attachments — upload PNG/JPG/PDF files (max 5MB, up to 3 files).

Click Save Changes to apply. The cell in the main table updates immediately.

Permission You can only edit nodes assigned to your group. If you see "Read-only mode", ask an admin to add you to the appropriate group.

5.2 Timeline

Below the edit form is the Timeline, showing the history of changes for this issue.

  • By default, it shows entries for the current node only (plus general comments and meeting notes).
  • Click View All to see the entire issue's history across all nodes.
  • Use the type dropdown to filter by State Changes, Comments, or Meeting Notes.

5.3 Adding Comments & Meeting Notes

  • Click + General Comment to add a general comment (with optional attachments).
  • Click + Meeting Note to add a meeting note tagged with a specific week.

6. Closed Page

Displays all closed issues, grouped by week (same layout as Tracker). Accessible without login (read-only).

  • Use the search bar to find specific closed issues.
  • Each cell shows the final state, check-in date, and note (same format as Tracker).
  • The Closed column shows the date the issue was closed.
  • The Note column shows the closing note.
  • Pagination is available at the bottom (50 per page).

The dashboard provides an overview of the project status. Login required.

  • Topic Cumulative Trend — stacked bar chart showing TBD/Dev/UAT/Close counts over weeks.
  • Closing Rate Trend — line chart of closing rate over time. If admin has entered manual trend data, it uses that data. Use the dropdown to switch between YTD, Last 30 weeks, or Last 50 weeks.

  • Almost Done — issues with only 1-2 nodes remaining.
  • Stale Topics — issues not updated in 90+ days.
  • Per-Node Stats — red line, UAT, and TBD counts broken down by node.

8. Calendar

A monthly view showing check-in dates. Each cell in the calendar shows which issues/nodes have a check-in date on that day. Navigate between months using the arrows. Login required.

9. Dark Mode

Click the ☾/☀ button in the navbar to toggle between Light and Dark mode. Your preference is saved in the browser and persists across sessions.

Tip Dark mode uses a low-contrast palette designed to reduce eye strain during extended use.

10. Super User Features

Super users (admins) have additional capabilities. If you are a super user, an Admin link appears in the navbar.

10.1 Create Issue

Click + New Issue in the Tracker toolbar, or go to Admin > New Issue. Fill in the required fields (#, Topic, Week) and submit.

10.2 Edit Issue (Topic / Path / Owner)

Open the side panel by clicking any cell, then click Edit Owner / Topic / Path to expand the edit section. Each field has its own save button.

10.3 Delete Issue

In the side panel, click the red Delete button. This is a soft delete — the issue is hidden but can be restored by an admin.

10.4 Close & Reopen

  • When all nodes are Done/Unneeded, a ✓ Close button appears next to the issue number. Click it, optionally add a note, and confirm.
  • On the Closed page, super users can click Reopen to move an issue back to ongoing.
  • Super users can also edit the Closed Note inline by clicking the Note cell on the Closed page.

10.5 Row Apply

Click the ✎ (pencil) icon next to an issue number to enter row-edit mode. Select which nodes to update, choose a state, and click Apply to update all selected nodes at once.

10.6 Meeting Mode

Click the wkXXXX Meeting Mode button in the Tracker toolbar. This opens a dedicated page for the current week where you can enter meeting notes for each issue in bulk.

11. FAQ

Q: I can't edit any cells. Why?

Your account must be active (approved by admin), and you must be assigned to a group that has permission for that node. Ask your admin to check your group assignments.

Q: How do I copy a Path?

Click the Path cell in the Tracker table. The text is automatically copied to your clipboard and a green "✓ Copied" toast appears briefly.

Q: What does the yellow highlight on a cell mean?

It means the cell was updated since your last visit. Click Mark All Read in the toolbar to clear the highlights.

Q: How does the Gitea issue link work?

If your admin has configured URL mappings, the issue number (#) in the Tracker table becomes a clickable link that opens the corresponding Gitea issue in a new tab. The mapping rules are:

  • P260 → prefix P, opens the perl_scripts repo issue #260
  • CN49 → prefix CN, opens the CN_TEAM repo issue #49
  • SNPS61 → prefix SNPS, opens the Synopsys repo issue #61
  • 301 → pure number, opens the QC_requests repo issue #301

Q: Can I export data?

Yes. Click Export Excel in the Tracker toolbar to download an .xlsx file with all ongoing and on-hold issues.

Q: The page looks different / broken after an update.

Press Ctrl + Shift + R to force-refresh and clear the browser cache.
