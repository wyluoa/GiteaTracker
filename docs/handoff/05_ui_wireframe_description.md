# 05 — UI Wireframe 文字描述

> 這份文件是 `04_ui_wireframe.html` 的純文字版本,給沒有視覺處理能力的 AI 工具或開發者參考。如果你能看 HTML/瀏覽器,直接看 `04_ui_wireframe.html` 會更直觀。

## 顏色系統

| 狀態 | 顯示文字 | 顏色 (hex) | 顏色描述 | 語意 |
|---|---|---|---|---|
| Done | `Done` | `#27ae60` | 深綠 | 程式已上線 |
| UAT done | `UAT✓` | `#3498db` | 藍 | 測試完成但還沒上線,等批次上線 |
| UAT | `UAT` | `#e67e22` | 橘 | 測試中 |
| Developing | `Dev` | `#95a5a6` | 灰 | 開發中 |
| TBD | `TBD` | `#8e44ad` | 紫 | 未決定 / 等外部因素 |
| Unneeded | `—` | `#bdc3c7` | 淺灰 | 這個 node 不需處理 |
| (空) | `·` | `#ecf0f1` | 很淺灰 | 還沒填 |

**特殊狀態樣式**:
- 紅線以上的 `UAT` 或 `TBD`:狀態色塊背景改為紅色 `#c0392b`(只改色塊,不改 cell 整格背景)
- 自上次查看以來有變動的 cell:cell 背景為黃色 `#fff9c4`
- 「Ready to Close」題目:題號旁邊加綠色 ✓ 標記

### Cell 內部結構

每個 node cell 內部從上到下:
1. **狀態色塊**(必須):彩色背景的 inline-block,顯示狀態文字
2. **Check-in 日期**(可選,小字 9px):藍灰色,例如 `→ 5/20` 或 `5/20`
3. **Short note**(可選,小字 9px,棕色斜體):例如 `等廠商 3/15 回覆`
4. **Badge**(可選,右上角):藍色小圓角,顯示該 cell 的 timeline 更新筆數,例如 `2`

### 互動

- **點擊 cell** → 開啟 Side Panel(Page 2)
- **點擊週次 header(▾/▸)** → 收合 / 展開該週的所有題目
- **勾選 checkbox** → 出現批次操作列
- **點擊題號** → 開啟該題的 side panel(預設選第一個 node)
- **滑鼠 hover cell** → 顯示 tooltip(簡單顯示最後更新時間 + 更新者)

### 排序

由舊到新(較舊的週次在上面,本週在下面)。理由:舊題目需要被看到、被處理;新題目可以等。

### 紅線

從 `settings.red_line_week_year` + `settings.red_line_week_number` 讀取。在對應週次之後插入一條紅色虛線分隔列。線之上(較舊)的所有題目中,`UAT` 或 `TBD` 狀態的 cell 套用紅色狀態色塊。

### On Hold 區塊

主畫面最下方的紅色背景區塊,顯示 `⏸ On Hold (N) ▸`,預設收合。點擊展開後顯示所有 `status = on_hold` 的題目,結構與 Ongoing 同。

## 頁面 2:Side Panel(編輯與 Timeline)

**觸發**: 在主畫面點擊 cell
**載入方式**: HTMX 動態載入 partial,不是整頁刷新
**位置**: 螢幕右側滑出,寬度約 580px


### Timeline 三種類型

1. **state_change**(藍色 ◐ 圖示)
   - 自動產生,當有人改 cell 的 state / check_in_date / short_note 時
   - 顯示「誰把哪個 node 從什麼改成什麼 + 時間」
   - 可附帶「更新說明」(填表時可選填)
   - 附件也歸屬於這筆 entry

2. **comment**(💬 圖示)
   - 一般留言,純文字
   - 用於非狀態變更的補充討論
   - 可附附件

3. **meeting_note**(📋 圖示,黃色背景區塊)
   - 會議紀錄,標記會議週次
   - 用於記錄會議當下的討論重點
   - 可附附件

### 篩選

下拉選單可只顯示某種類型的 entry:全部 / 只看狀態變更 / 只看留言 / 只看會議紀錄。

## 頁面 3:會議模式 (Meeting Mode)

**觸發**: 主畫面 toolbar 上的「📋 進入 wkXXX 會議模式」按鈕
**Route**: `/meeting/<week_year>/<week_number>`

### 用途

開會時連續記錄多題的會議紀錄,不用一題一題開 side panel。送出後系統自動把每筆分散存到對應題目的 timeline。

```
├────────────────────────────────────────────────────────────┤
│ #305  | title7              | [textarea: 會議重點...]      │
│       | Owner: Ivy          | Ivy 報告 A14 已完成,N3 預計  │
│       | Requestor: Frank    | 下週 check in...             │
├────────────────────────────────────────────────────────────┤
│ #306  | another topic       | [textarea: 會議重點...]      │
│       | Owner: Stan         |                              │
│       | Requestor: Grace    |                              │
```

### 邏輯

- 預設列出該週次的所有題目,可手動加減

- 一次送出後跳回主畫面

### 卡片內容(每個 active node 一張)

最顯眼:**紅線以上未完成數**(28px 大字 + 紅色)

### 特殊卡片

- **Ready to Close**(黃底):所有 node 都已完成的題目數量。點擊跳到列表,super user 可手動關單
- **On Hold**(紅底):暫停中題目數量

## 頁面 6:Closed 分頁

**Route**: `/closed`

跟主畫面類似的表格,但只列已關單的題目。

差異:
- 預設只載入最近 N 筆,要看更舊的需要往前翻頁或搜尋
- 所有 cell 唯讀
- Super user 多一個「↺ Reopen」按鈕,點擊後彈出 modal 要求填寫理由(會寫入 timeline + audit_log),確認後題目回到 Ongoing
- 支援搜尋 / 篩選

## 頁面 7:Admin 後台

**Route**: `/admin`(super user only)

包含以下子頁:

### 7.1 待審帳號 (`/admin/pending_users`)
列出 `status = pending` 的使用者,每筆有「Approve」「Reject」按鈕。Approve 時可順便指派 group。

### 7.2 Users (`/admin/users`)
所有使用者列表,可改 display_name、改 group、停用、設為 super user。

### 7.3 Groups (`/admin/groups`)
- 建立 / 改名 / 刪除 group
- 指派成員(checkbox 列出所有 active user)
- 指派可編輯的 nodes(checkbox 列出所有 active node)

### 7.4 Nodes (`/admin/nodes`)
- 建立 node(填 code + display_name + sort_order)
- 改 display_name 和 sort_order
- 停用 node(`is_active = 0`,不顯示在主畫面但歷史保留)

### 7.5 紅線設定 (`/admin/red_line`)
- 選擇 ISO 年 + 週次
- 顯示目前紅線位置
- 改動會寫入 audit_log

### 7.6 SMTP 設定 (`/admin/smtp`)
- mail server host / port / username / password / from email
- 寄送測試信功能

### 7.7 Audit Log (`/admin/audit`)
時間倒序列出所有 audit_log 紀錄,可篩選 actor / action / target。

## 頁面 8:登入 / 註冊 / 忘記密碼

### 8.1 登入 (`/login`)
- username + password
- 「忘記密碼?」連結
- 「沒有帳號?點此註冊」連結

### 8.2 註冊 (`/register`)
- username, email, display_name, password (兩次)
- 註冊後顯示「您的帳號已建立,等待管理員審核。審核通過後您會收到通知信。」

### 8.3 忘記密碼 (`/forgot_password`)
- 輸入 email
- 系統寄 reset link 到信箱
- 顯示「如果該 email 存在,我們已寄送密碼重設連結」(不洩漏 email 是否存在)

### 8.4 重設密碼 (`/reset_password/<token>`)
- 從 URL 帶 token,系統驗證
- 輸入新密碼兩次
- 驗證通過後標記 token 為 used、更新 password_hash、跳轉到登入頁

## 通用 UI 規範

### 按鈕顏色

- **主要動作**(藍色 `#3498db`):儲存、登入、套用
- **次要動作**(灰色 `#95a5a6`):取消、標記已讀、匯出
- **警示動作**(橘色 `#f39c12`):進入會議模式、Ready to Close
- **危險動作**(紅色 `#c0392b`):刪除、Reopen
- **成功動作**(綠色 `#27ae60`):批次套用、確認上線

### 字體

預設無襯線(系統 sans-serif),包含中文字型 fallback (`-apple-system, "Noto Sans TC", sans-serif`)。

### Spacing

緊湊型 UI,table cell padding 約 4-6px。內網工具優先資訊密度,而非美觀的留白。

### 響應式

不需要支援手機。最低支援 1280x800 桌面螢幕。
